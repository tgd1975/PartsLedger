# PIC16F628 / PIC16F628A — 18-pin PIC microcontroller family

18-pin DIP. 2 K words of flash, 224 bytes RAM, 128 bytes EEPROM, 16 I/O pins, USART, two analog comparators, two 8-bit timers and one 16-bit timer, one CCP (capture/compare/PWM). The bigger sibling of the [PIC12F675](pic12f675.md) — more pins, more flash, and a hardware UART. Supply 3.0–5.5 V (A) or 4.0–5.5 V (pre-A).

This page covers both variants in this inventory: the current **PIC16F628A** and the legacy **PIC16F628** (pre-A, e.g. `PIC16F628-20I/P`). They are pin-compatible and share the same instruction set — most projects don't care which one you reach for. See the [Differences](#differences-pic16f628a-vs-pic16f628-pre-a) section for what changed in the A revision.

Datasheet (A): [DS40044G](https://ww1.microchip.com/downloads/en/DeviceDoc/40044G.pdf)
Datasheet (pre-A): [DS40300C](https://ww1.microchip.com/downloads/en/DeviceDoc/40300c.pdf)
Octopart: [PIC16F628A](https://octopart.com/search?q=PIC16F628A) · [PIC16F628](https://octopart.com/search?q=PIC16F628)

## ELI5

Same idea as the [PIC12F675](pic12f675.md) — a tiny computer on a chip, programmed once, runs forever — but with more of everything:

- **More pins**: 16 I/O lines instead of 5. Enough for an LCD, a keypad, or a handful of outputs without external shift registers.
- **More memory**: 2 K of program flash instead of 1 K. Still small, but enough for a serial protocol, some state machines, or a menu system.
- **Built-in UART**: it can talk serial (RX/TX) in hardware, so you don't have to bit-bang. This is the most common reason to pick a 16F628 over a 12F675.

What it still can't do: USB, network, audio, anything that needs much memory or fast math. For those, jump up to a PIC18, an AVR, or an STM32.

The "A" is just a revision. If you have a 16F628 from before ~2004, it's the original; if it has an "A" on the marking, it's the 2004+ revision with a couple of nice-to-haves (internal oscillator, programmable brown-out). For hobbyist use they behave the same.

## At a glance

- **Flash**: 2048 × 14-bit words (~200 lines of C, less for ambitious code)
- **RAM**: 224 bytes
- **EEPROM**: 128 bytes (persists across power cycles; ~100k write endurance)
- **I/O pins**: 16 total — PORTA (8 pins, RA0–RA7) and PORTB (8 pins, RB0–RB7)
- **USART**: 1 (hardware UART; RX on RB1, TX on RB2)
- **Comparators**: 2, with programmable voltage reference (VREF on RA2)
- **Timers**: TMR0 (8-bit), TMR1 (16-bit, can use external 32 kHz crystal), TMR2 (8-bit with prescaler+postscaler)
- **CCP**: 1 (capture / compare / PWM, on RB3)
- **Clock**: external up to 20 MHz; A also has 4 MHz internal RC (factory-calibrated)
- **Supply**: 3.0–5.5 V (A) / 4.0–5.5 V (pre-A)
- **Programming**: ICSP via 5 pins
- **No ADC.** If you need analog inputs, you have the two comparators only — for true ADC, look at the PIC16F88 or a PIC18 part.

## Pinout (DIP-18, top view)

```text
              ┌──U──┐
RA2/AN2/VREF 1│     │18 RA1/AN1
     RA3/AN3 2│     │17 RA0/AN0
   RA4/T0CKI 3│     │16 RA7/OSC1
    RA5/MCLR 4│     │15 RA6/OSC2
         VSS 5│     │14 VDD
     RB0/INT 6│     │13 RB7/PGD
      RB1/RX 7│     │12 RB6/PGC
      RB2/TX 8│     │11 RB5
    RB3/CCP1 9│     │10 RB4/PGM
              └─────┘
```

| Pin | Name | Roles                                                                                |
| --- | ---- | ------------------------------------------------------------------------------------ |
| 1   | RA2  | Digital I/O / comparator AN2 / **VREF** output                                       |
| 2   | RA3  | Digital I/O / comparator AN3 / comparator 1 output                                   |
| 3   | RA4  | Digital I/O / Timer0 external clock (T0CKI) / comparator 2 output. **Open-drain.**   |
| 4   | RA5  | **Input-only** / MCLR reset / programming VPP (~13 V during write)                   |
| 5   | VSS  | Ground                                                                               |
| 6   | RB0  | Digital I/O / external interrupt (INT)                                               |
| 7   | RB1  | Digital I/O / **USART RX** / synchronous DT                                          |
| 8   | RB2  | Digital I/O / **USART TX** / synchronous CK                                          |
| 9   | RB3  | Digital I/O / CCP1 (capture / compare / PWM)                                         |
| 10  | RB4  | Digital I/O / low-voltage programming entry (PGM)                                    |
| 11  | RB5  | Digital I/O                                                                          |
| 12  | RB6  | Digital I/O / Timer1 osc output (T1OSO) / Timer1 clock input (T1CKI) / **ICSPCLK**   |
| 13  | RB7  | Digital I/O / Timer1 osc input (T1OSI) / **ICSPDAT**                                 |
| 14  | VDD  | Power                                                                                |
| 15  | RA6  | Digital I/O / oscillator output (OSC2/CLKOUT)                                        |
| 16  | RA7  | Digital I/O / oscillator input (OSC1/CLKIN)                                          |
| 17  | RA0  | Digital I/O / comparator AN0                                                         |
| 18  | RA1  | Digital I/O / comparator AN1                                                         |

A few quirks worth knowing:

- **RA4 is open-drain.** When you set it high in code, it floats — needs a pull-up resistor to drive an LED or anything else.
- **RA5 is input-only** when MCLR is disabled (config bit MCLRE = OFF). Always input-only otherwise as the MCLR pin.
- **RA6 and RA7 are I/O only when the internal oscillator is selected**; in HS/XT/LP modes they become OSC2/OSC1 and are unavailable.

## Sample circuit — minimum to run a program

To get a programmed chip blinking an LED on RB0, using the A's internal oscillator (PIC16F628A only; pre-A needs external crystal — see [Variations](#variations)):

- **Pin 14 (VDD)** → +5 V
- **Pin 5 (VSS)** → GND
- **100 nF decoupling cap** between pins 14 and 5, close to the chip
- **Pin 4 (MCLR)** → 10 kΩ pull-up to VDD (or disable MCLR in config bits to free RA5 as an input)
- **Pin 6 (RB0)** → 330 Ω → LED anode; LED cathode → GND
- All other I/O pins → leave open if unused

Config bits: set oscillator mode to INTOSC, disable MCLR if RA5 is needed as input, disable LVP, disable WDT for simple test code.

## Programming setup

ICSP via 5 pins (PICkit 3/4/5 or clone):

| Programmer pin | Chip pin       |
| -------------- | -------------- |
| VPP / MCLR     | Pin 4 (RA5)    |
| VDD            | Pin 14         |
| VSS / GND      | Pin 5          |
| PGD (data)     | Pin 13 (RB7)   |
| PGC (clock)    | Pin 12 (RB6)   |

Different ICSP pin numbers from the PIC12F675 — easy to mix up when switching between the two. RB6/RB7 are commonly used for non-ICSP roles too; isolate the programmer with series resistors if they're connected to other circuitry.

Toolchain: **MPLAB X IDE + XC8 compiler** (free from Microchip).

## Variations

- **External crystal** (more accurate than the internal RC, and required on the pre-A which has no internal osc) → 4–20 MHz crystal on pins 15 and 16, with two 15–22 pF caps to GND; config bits set to XT or HS mode. Costs you RA6 and RA7.
- **32 kHz watch crystal on Timer1** → connect to pins 12 and 13 (T1OSI/T1OSO) for real-time-clock applications. Costs you RB6 and RB7 (and ICSP — program first, then wire the crystal).
- **Hardware UART** → RB1 (RX) and RB2 (TX), 9600–115200 baud typical at 4 MHz. Frees you from bit-banging serial.
- **PWM output** → CCP1 on RB3, typically 8–10 bit resolution at audio-ish frequencies.
- **Free RA5 as input** → disable MCLR in config bits. RA5 then becomes a digital input (still no output drive).

## Differences: PIC16F628A vs. PIC16F628 (pre-A)

The original PIC16F628 was superseded by the A revision around 2004. Microchip discontinued the non-A; the parts in this inventory are leftover stock. For most purposes the two are interchangeable, but a few things changed:

| Feature                | PIC16F628 (pre-A) | PIC16F628A                              |
| ---------------------- | ----------------- | --------------------------------------- |
| Internal RC oscillator | none              | 4 MHz, factory-calibrated               |
| Brown-out reset        | fixed threshold   | programmable, can be disabled           |
| Supply range           | 4.0–5.5 V         | 3.0–5.5 V                               |
| Active-mode current    | higher            | lower (~half typical; verify datasheet) |
| Configuration bits     | basic             | more granular (extra fuses)             |
| Microchip availability | obsolete          | active                                  |

For hobbyist projects, the practical implications:

- **Pre-A needs an external clock source.** Either a crystal or an RC network. No "just power it and go".
- **Pre-A at 3 V doesn't work.** Stick to 5 V for the legacy parts; the A can run from 3.3 V.
- **Programs from A often run unchanged on pre-A** — as long as you set the oscillator config correctly and don't rely on programmable BOR.
- The instruction set, memory map, and peripheral set are otherwise identical.

If you're starting a new design today, use the A. Reach for the pre-A only to draw down old stock.

## Watch out for

- **RA4 is open-drain** — needs a pull-up to drive anything high. Trips up first-time users.
- **RA5 is input-only.** Whether MCLR is enabled or not, you cannot drive an output from this pin.
- **OSCCAL** (internal-oscillator calibration) lives at the last flash address on the A. Some older programmers erase it on a bulk erase; lose it and the internal RC drifts. Modern tools preserve it by default.
- **Configuration bits matter a lot.** Wrong oscillator mode is the most common reason a freshly-programmed chip seems dead. Always set config bits explicitly in code.
- **LVP (low-voltage programming) defaults to enabled** in some toolchain configs. If you don't intend to use LVP, disable it — otherwise RB4 is hijacked as the LVP entry pin and behaves oddly.
- **No ADC.** This is the part you'll forget about until you reach for an analog input and find only two comparators.
- **Pre-A is obsolete.** Don't design new boards around it.

## Pairs naturally with

- **MCP4922** if you need analog output — the 16F628 has no DAC, but it can drive the MCP4922 over SPI (bit-banged or via the USART in synchronous mode).
- **74HC595 / 74HC165** for more pins when 16 isn't enough.
- **ULN2803** to sink current for relays or solenoids beyond the chip's per-pin ~25 mA.
- The [PIC12F675](pic12f675.md) for the same kind of job in a smaller package, when 8 pins are enough.
- A separate **ICSP programmer** (PICkit) — required, not in this inventory.
