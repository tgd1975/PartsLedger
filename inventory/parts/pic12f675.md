# PIC12F675 — 8-pin PIC microcontroller

8-pin DIP. 1 K words of flash, 64 bytes RAM, 128 bytes EEPROM, 6 I/O pins (one input-only), 10-bit ADC on 4 channels, one comparator, two timers. Runs at up to 20 MHz with an external crystal, or 4 MHz on the internal oscillator. Supply 2.0–5.5 V.

Datasheet: [DS41190F](https://ww1.microchip.com/downloads/en/DeviceDoc/41190F.pdf)
Octopart: [PIC12F675](https://octopart.com/search?q=PIC12F675)

## ELI5

It's a tiny computer on an 8-pin chip. You write a program on your laptop, send it to the chip once with a small device called a programmer (PICkit), and from then on the chip runs that program every time it gets power.

What it can do:

- Read up to 4 voltages (e.g. from a pot, a sensor, a battery divider) — that's the ADC
- Switch up to 5 things on and off (LEDs, a relay coil via a driver, a buzzer) — that's the digital outputs
- Notice when a button is pressed and do something about it
- Keep time (seconds, milliseconds) — that's the timers
- Remember settings even when the power is off — that's the EEPROM

What it can't do: USB, network, screen output, audio, anything that needs much memory or fast math. For those, reach for a bigger chip (a bigger PIC, an AVR, an STM32, an ESP32). The PIC12F675 is for small, fixed, low-pin-count jobs where an Arduino would be overkill and physically too big.

The "ICSP programmer" is the only extra hardware you really need to start — without it the chip is just an inert 8-pin package. <!-- lint: ok -->

## At a glance

- **Flash**: 1024 × 14-bit words (small — think ~100 lines of C, less for ambitious code)
- **RAM**: 64 bytes
- **EEPROM**: 128 bytes (persists across power cycles; ~100k write endurance)
- **I/O pins**: 6 total — GP0–GP2, GP4, GP5 are bidirectional; **GP3 is input-only**
- **ADC**: 10-bit, 4 channels (AN0–AN3)
- **Comparator**: 1
- **Timers**: TMR0 (8-bit), TMR1 (16-bit, with optional external clock)
- **Clock**: 4 MHz internal RC (factory-calibrated); or external up to 20 MHz
- **Supply**: 2.0–5.5 V
- **Programming**: ICSP (in-circuit serial programming) via 5 pins

## Pinout (DIP-8, top view)

```text
                ┌──U──┐
           VDD 1│     │8 VSS
GP5/OSC1/CLKIN 2│     │7 GP0/AN0/CIN+/ICSPDAT
  GP4/AN3/OSC2 3│     │6 GP1/AN1/CIN-/ICSPCLK
  GP3/MCLR/VPP 4│     │5 GP2/AN2/T0CKI/INT
                └─────┘
```

| Pin | Name | Roles                                                              |
| --- | ---- | ------------------------------------------------------------------ |
| 1   | VDD  | Power, 2.0–5.5 V                                                   |
| 2   | GP5  | Digital I/O / external osc input / Timer1 clock input              |
| 3   | GP4  | Digital I/O / ADC channel AN3 / external osc output                |
| 4   | GP3  | **Input-only** / MCLR reset / programming VPP (~13 V during write) |
| 5   | GP2  | Digital I/O / ADC AN2 / Timer0 clock / external interrupt          |
| 6   | GP1  | Digital I/O / ADC AN1 / comparator input / **ICSPCLK**             |
| 7   | GP0  | Digital I/O / ADC AN0 / comparator input / **ICSPDAT**             |
| 8   | VSS  | Ground                                                             |

## Sample circuit — minimum to run a program

To get a programmed chip blinking an LED on GP0:

- **Pin 1 (VDD)** → +5 V (or anything 2.0–5.5 V)
- **Pin 8 (VSS)** → GND
- **100 nF decoupling cap** between pins 1 and 8, close to the chip
- **Pin 4 (MCLR)** → 10 kΩ pull-up to VDD (or disable MCLR in config bits to free GP3 as an input)
- **Pin 7 (GP0)** → 330 Ω → LED anode; LED cathode → GND
- All other GP pins → leave open if unused, or pull to a defined level

That's the whole circuit. The internal RC oscillator means no crystal is needed for most jobs.

## Programming setup

You need an ICSP programmer (e.g. PICkit 3/4/5, or a clone) connected to:

| Programmer pin | Chip pin     |
| -------------- | ------------ |
| VPP / MCLR     | Pin 4 (MCLR) |
| VDD            | Pin 1        |
| VSS / GND      | Pin 8        |
| PGD (data)     | Pin 7 (GP0)  |
| PGC (clock)    | Pin 6 (GP1)  |

If GP0 and GP1 are also wired into your circuit, make sure their loads don't fight the programmer (typical fix: small series resistor or isolation).

Toolchain: **MPLAB X IDE + XC8 compiler** (free from Microchip). Programs in C or assembly.

## Variations

- **External crystal** (more accurate frequency than internal RC) → 4–20 MHz crystal on pins 2 and 3, with two 15–22 pF caps to GND; config bits set to XT/HS osc mode. Costs you GP4 and GP5.
- **Free GP3 as an input** → disable MCLR in config bits (MCLRE = OFF). Now GP3 is an input-only digital pin. Trade-off: no external hardware reset.
- **Sleep mode for battery** → put the chip to sleep between events; wake on interrupt or watchdog. Drops current to microamps.
- **Read a pot** → wire the pot's wiper to GP0/1/2/4 (any AN pin), full-scale between VDD and VSS; read the ADC in code.

## Watch out for

- **GP3 is input-only.** Easy to forget when planning pin assignments.
- **OSCCAL calibration word** lives at the last flash address. Some old programmer settings erase it on a bulk erase — losing it means the internal RC drifts. Modern tools preserve it by default, but verify.
- **Configuration bits matter a lot.** MCLR enable, oscillator mode, brown-out, code protect — wrong values can leave the chip mute or unprogrammable. Always set them explicitly in code. <!-- lint: ok -->
- **ICSP pins double as I/O.** Heavy loads on GP0/GP1 can prevent in-circuit programming. Keep them lightly loaded or add series resistors.
- **Legacy part.** Still available and well-documented, but newer 8-pin PICs (PIC12F1822, PIC16F18313) have more flash, better peripherals, and the same price. Worth knowing about if starting a new design rather than using on-hand stock.
- **No hardware UART, no I²C, no SPI.** Anything serial has to be bit-banged in software, which eats both flash and CPU time.

## Pairs naturally with

- **74HC595** if 5 outputs aren't enough — shift in a byte over 3 pins, get 8 more.
- **74HC165** if 5 inputs aren't enough — same trick in reverse.
- **ULN2803** to drive relays, solenoids, or anything inductive from the chip's outputs (the GP pins can only sink/source ~25 mA each).
- A separate **ICSP programmer** (PICkit) — not in this inventory, but required to put code on the chip.
